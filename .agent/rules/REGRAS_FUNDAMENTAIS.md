# REGRAS FUNDAMENTAIS - OBRIGATÓRIO LER

## ⚠️ REGRA DE OURO

**VOCÊ É TERMINANTEMENTE PROIBIDO DE FAZER QUALQUER ALTERAÇÃO, COMMIT OU DEPLOY SEM APROVAÇÃO EXPLÍCITA DO USUÁRIO.**

### Isso significa:
- ❌ NÃO fazer alterações de código
- ❌ NÃO fazer commits
- ❌ NÃO fazer deploy
- ❌ NÃO reverter mudanças

### O que FAZER:
- ✅ SEMPRE mostrar ao usuário ANTES de fazer qualquer mudança
- ✅ ESPERAR aprovação explícita do usuário
- ✅ Se o usuário não aprovar, NÃO fazer nada
- ✅ Se o usuário reclamar de algo feito sem aprovação, reverter APENAS se ele pedir

### Fluxo correto:
1. Usuário dá uma ordem
2. Você planeja e mostra o plano
3. Usuário aprova
4. Você executa
5. Usuário verifica e aprova o resultado
6. SOMENTE então faz commit e deploy

---

*Esta regra é ABSOLUTA e não tem exceção.*
